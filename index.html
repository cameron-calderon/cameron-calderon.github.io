Zhang Suen Implementation

This project was focused on understanding the definition of the Jaccard measure of similarity, or the Jaccard coefficient.Ultimately I was able to familiarize with the Sokoto Coventry Fingerprint Dataset by reading Shehu et al.’s paper. Check out my code on my linked GitHub.

My observations from the score distribution plots is the less the search area the closer the genuine and imposter scores are together. The d prime score varies from plot to plot. But from my understanding the bigger the search area the more accurate data can be recovered.

Plot with a search area of 15 resulted in a D-Prime of .56.

Plot with a search area of 20 resulted in a D-Prime of .40.

Pain Classification

One way of managing soldier’s pain is identifying and treating pain as soon as it occurs. I was interested in ways to automatically determine if soldiers are in pain (e.g. wounded), in real-time. I hoped to develop a system that can identify(i.e. classify) pain from physiological data that will be collected from wearable devices. The data via CSV was collected to train and test the system.For the output, I provide the accuracy and confidence of the system.

I calculated the mean, variance, entropy, min, and max for all 4 data types given. In total there were 20 features (5 features from each type). These values were my training and testing data. For the 4 data types, I had a list of size 5 for each instance of the data. For the fusion, I created a list of size 20 that contains all data types.Then, using these features to build and train a random forest to classify pain vs. no pain.

For training and testing, I performed 10-fold cross-validation. Each fold had 6 subjects in it using 9 of them for training and 1 for testing. The output printed the confusion matrix, classification accuracy, precision, and recall. Finally, doing 10-fold cross validation, I took the average of these values over all folds of the testing. For the confusion matrix, I had 10 confusion matrices from each test, I added these matrices and took the average for the final matrix.

The fusion worked from respiration and Systolic because it integrated multiple sources to conclude a comprehensive result. While it did do that for all metrics, these two specifically have more of a correlation to pain, therefore utilizing fusion even more so. On the contrary, using fusion for the metrics that don’t categorize pain as much, fusion cannot be used to its fullest extent.

The systolic seemed to have the most variability. This seems so because, as mentioned in question 1, it is associated most with pain. This metric, according to pain, will vary greatly when being tested, hence the ebbs and flows.

The full code is on my GitHub
